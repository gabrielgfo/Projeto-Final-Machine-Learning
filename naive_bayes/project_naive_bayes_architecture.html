Naive Bayes Implementation Architecture

Apresentação da arquitetura real do pipeline de Naive Bayes para classificação de ALTO_RISCO_EVASAO em cursos do Censo INEP 2024.

  • models/naive_bayes/model_factory.py — fábrica de variantes
  • models/naive_bayes/preprocessing.py — 3 pipelines por variante
  • models/naive_bayes/train.py — 90 experimentos + CV 5-fold
  • data/preprocessamento.py — get_df_preprocessado()
  • analysis/audit/evaluator.py — reavaliação unificada com ComplementNB

Vencedor empírico do projeto: ComplementNB com alpha=0.16121212121212125, norm=True

Slide 2 — A Estrutura no Código

O projeto não implementa Naive Bayes do zero: encapsula o sklearn em create_model(config), escolhendo a variante via dicionário de configuração após busca comparativa em train.py.

Atributos internos (sklearn, pós-fit)

Slide 3 — A Fase de Treino (fit)

Fluxo real em train.py → _avaliar_configuracao() e _salvar_melhor_modelo():

  1. carregar_dados() → get_df_preprocessado() (features numéricas + alvo binário)
  2. StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
  3. get_preprocessing_pipeline(model_type) + transformar_dados(..., fit=True)
  4. aplicar_smote_treino(X_train_proc, y_train) — só no treino do fold
  5. create_model(config) → modelo.fit(X_train_bal, y_train_bal)

X_train_bal, y_train_bal = aplicar_smote_treino(X_train_proc, y_train)

modelo = create_model(config)
modelo.fit(X_train_bal, y_train_bal)
ComplementNB (variante vencedora) — suavização Laplace com \(\alpha = 0.16121212121212125\): \[ \hat\theta_{c,j} = \frac{\alpha + \sum_i x_{i,j}\,\mathbb{1}[y_i=c]}{\alpha \cdot d + \sum_{i,j} x_{i,j}\,\mathbb{1}[y_i=c]} \] Priori por contagem (em log, dentro do sklearn): \[ \log \hat\pi_c = \log \frac{N_c + \alpha}{N + K\alpha} \]
data/preprocessamento.py 
def get_df_preprocessado():
    df = _carregar_dados_brutos()
    df = _calcular_alvo(df)  # taxa_evasao ≥ mediana
    return _selecionar_features(df)

Filtro populacional: QT_ING >= 10. Alvo: alto_risco_evasao quando taxa_evasao >= mediana(taxa_evasao). Features: apenas colunas numéricas, excluindo identificadores e vazamento (COLUNAS_VAZAMENTO, prefixos QT_MAT, QT_CONC).

Para ComplementNB, o pipeline aplica apenas MinMaxScaler() — escala features para \([0,1]\), requisito da variante complementar com contagens não negativas.

Slide 4 — A Fase de Predição (predict)

Após transformar o teste com o mesmo pipeline ajustado no fold:

models/naive_bayes/train.py — _avaliar_configuracao() 
y_pred = modelo.predict(X_test_proc)
y_proba = modelo.predict_proba(X_test_proc)[:, 1]

Fluxo interno (sklearn ComplementNB)

O projeto delega a decisão ao sklearn, que calcula \(\arg\max_c \big[\log \hat\pi_c + \sum_j x_j \cdot f_{c,j}\big]\) onde \(f_{c,j}\) vem de feature_log_prob_ (forma complementar com norm=True).

predict() retorna a classe com maior log-posterior (0 ou 1). predict_proba() expone probabilidades via np.exp(log_prob - logsumexp(log_prob)) — o projeto usa apenas a coluna da classe positiva para ROC-AUC.

Critério de seleção entre 90 configs: maior recall médio → desempate por F1 → ROC-AUC → menor recall_std.

Slide 7 — Por que ComplementNB é ótimo para MEUS dados

  • Formato tabular misto codificado numericamente — get_df_preprocessado() entrega dezenas de features numéricas (contagens INEP + proporções derivadas), não texto bruto. ComplementNB lida bem com features não negativas após MinMaxScaler.
  • Desbalanceamento de classes — o alvo por mediana gera classes ~50/50, mas o pipeline aplica SMOTE(random_state=42) no treino; ComplementNB foi projetado para dados complementares/implementações com norm=True que estabilizam pesos em cenários assimétricos.
  • Seleção empírica, não assumida — entre 90 experimentos, ComplementNB superou GaussianNB (recall 0.955) e BernoulliNB (recall 0.878) no critério do projeto: maximizar recall de evasão.
  • Hiperparâmetro tunado — alpha=0.1612… evita zero absoluto em features raras do Censo sem suavização excessiva (grid amostrado em np.linspace(0.01, 5.0, 100)).
  • Velocidade e interpretabilidade operacional — fit/predict vetorizados no sklearn; artefato único (best_model.pkl) com preprocessor acoplado facilita deploy e comparação com LR, Árvore, SVM e MLP no mesmo repositório.

Conclusão: a implementação do projeto escolhe ComplementNB + MinMaxScaler + SMOTE no treino porque esse trio maximizou recall na validação cruzada estratificada sobre microdados INEP 2024 — exatamente o objetivo de alerta precoce de ALTO_RISCO_EVASAO.
